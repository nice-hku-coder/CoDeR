# /data/processed 数据文件来源与生成命令

本文档逐个说明 ` /home/xingkun/ant/data/processed ` 下每个 `.jsonl` 文件的来源、依赖关系与生成命令。

---

## 1) `train_triplets.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/train_triplets.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_triplets.py`
- **来源数据**：HuggingFace `snli` 数据集（`train` split）
- **生成逻辑**：
  - 从 NLI 样本中提取同一 `premise` 下的 `entailment(0)` 与 `contradiction(2)`
  - 构造成 `(query, positive, hard_negative)` 三元组
- **实际执行过的命令**：
```bash
python3 experiments/build_triplets.py --dataset snli --train-max 4000 --val-max 800
```

---

- **文件路径**：`/home/xingkun/ant/data/processed/val_triplets.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_triplets.py`
- **来源数据**：HuggingFace `snli` 数据集（`validation` split）

- **实际执行过的命令**：

## 3) `smoke_eval.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/smoke_eval.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_triplets.py`
- **来源数据**：脚本内置手工样例（非外部数据）
- **生成逻辑**：
  - 构造小规模 smoke test（hotel/peanut/price）
  - 每个 query 搭配 `satisfies=1/0` 的文档列表
- **实际执行过的命令**：
```bash
python3 experiments/build_triplets.py --dataset snli --train-max 4000 --val-max 800
```

## 4) `demo_corpus.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/demo_corpus.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_triplets.py`
- **来源数据**：由 `smoke_eval.jsonl` 派生
- **生成逻辑**：
  - 把 `smoke_eval` 里的文档扁平化并去重
  - 生成 `{id, text}` 形式的小语料库
- **实际执行过的命令**：
```bash
python3 experiments/build_triplets.py --dataset snli --train-max 4000 --val-max 800
```

---

## 5) `constraint_benchmark_v1.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/constraint_benchmark_v1.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_constraint_benchmark.py`
- **来源数据**：脚本内置模板与词表（非外部下载）
- **生成逻辑**：
  - 合成三类约束 query：
    - `negation`
    - `exclusion`
    - `numeric`
  - 每类生成指定数量样本，混合正负文档并打 `satisfies` 标签
- **实际执行过的命令**：
```bash
python3 experiments/build_constraint_benchmark.py \
  --output-file data/processed/constraint_benchmark_v1.jsonl \
  --num-negation 100 \
  --num-exclusion 100 \
  --num-numeric 100 \
  --seed 42
```

## 6) `retrieval_corpus_v1.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/retrieval_corpus_v1.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_retrieval_benchmark.py`
- **上游文件**：`constraint_benchmark_v1.jsonl`
- **生成逻辑**：
  - 从 benchmark 中提取所有文档文本
  - 全局去重后生成统一语料库 `{doc_id, text}`
- **实际执行过的命令**：
```bash
python3 experiments/build_retrieval_benchmark.py \
  --source-eval-file data/processed/constraint_benchmark_v1.jsonl \
  --output-corpus-file data/processed/retrieval_corpus_v1.jsonl \
  --output-benchmark-file data/processed/retrieval_benchmark_v1.jsonl
```

## 7) `retrieval_benchmark_v1.jsonl`

- **文件路径**：`/home/xingkun/ant/data/processed/retrieval_benchmark_v1.jsonl`
- **来源脚本**：`/home/xingkun/ant/experiments/build_retrieval_benchmark.py`
- **上游文件**：`constraint_benchmark_v1.jsonl`
- **生成逻辑**：
  - 为每个 query 生成统一评测结构：
    - `topical_relevant_doc_ids`
    - `constraint_satisfying_doc_ids`
    - `graded_relevance`
- **实际执行过的命令**：
```bash
python3 experiments/build_retrieval_benchmark.py \
  --source-eval-file data/processed/constraint_benchmark_v1.jsonl \
  --output-corpus-file data/processed/retrieval_corpus_v1.jsonl \
  --output-benchmark-file data/processed/retrieval_benchmark_v1.jsonl
```

## 8) `retrieval_triplets_v1.jsonl`

- **文件路径**：`/data/xingkun/CoDeR/data/processed/retrieval_triplets_v1.jsonl`
- **来源脚本**：临时转换脚本（从 `retrieval_benchmark_v1.jsonl` 和 `retrieval_corpus_v1.jsonl` 生成）
- **上游文件**：
  - `retrieval_benchmark_v1.jsonl`
  - `retrieval_corpus_v1.jsonl`
- **生成逻辑**：
  - `query` 直接来自 `retrieval_benchmark_v1.jsonl`
  - `positive` 从 `constraint_satisfying_doc_ids` 对应的文档文本中取
  - `hard_negative` 从同一 query 的 `topical_relevant_doc_ids` 中，排除满足约束的 doc 后取
  - 每个 query 生成所有 `positive x hard_negative` 组合
- **生成数量**：`2700` 条 triplet

## 数据血缘总览（简版）

```text
SNLI(train/validation)
  -> build_triplets.py
  -> train_triplets.jsonl / val_triplets.jsonl
  -> smoke_eval.jsonl -> demo_corpus.jsonl

build_constraint_benchmark.py (模板合成)
  -> constraint_benchmark_v1.jsonl
  -> build_retrieval_benchmark.py
  -> retrieval_corpus_v1.jsonl / retrieval_benchmark_v1.jsonl
  -> retrieval_triplets_v1.jsonl
```
